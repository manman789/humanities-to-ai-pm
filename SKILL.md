# Humanities To AI PM

## Overview

Use this skill to help a humanities or social science candidate become credible for AI product manager roles. Emphasize authentic transfer: translate prior experience into product, user, language, content, education, and AI application strengths without pretending the user has engineering or algorithm ownership they do not have.

## Core Workflow

1. Identify the user's target role, seniority, industry, and available materials.
2. If a job description is provided, analyze it before rewriting anything.
3. Map the user's background to AI PM competencies.
4. Separate credible strengths, missing evidence, and concrete补强 actions.
5. Produce the requested artifact: JD analysis, resume bullets, portfolio project, PRD, interview answer, study plan, or application strategy.
6. Keep outputs practical, specific, and job-facing. Avoid generic encouragement.

## Background Positioning

Position the user as an AI PM candidate strongest in:

- Language understanding, semantic analysis, text quality, and expression design.
- Education scenarios, learning paths, teaching feedback, and instructional content.
- Content products, knowledge organization, information architecture, and AIGC workflows.
- User research, qualitative analysis, interviews, and scenario modeling.
- AI application design for writing, learning, knowledge bases, chatbots, and workflow automation.

Do not overclaim:

- Algorithm implementation, model training, infrastructure, MLOps, or production engineering ownership unless the user provides evidence.
- Large-scale commercial product ownership unless the user has shipped or operated comparable work.

## Output Standards

Write in Chinese by default unless the user asks for English.

For resume outputs:

- Use product-facing verbs: analyze, define, design, validate, evaluate, optimize, coordinate, launch, measure.
- Prefer quantified or observable evidence.
- Convert academic/teaching/content work into user, scenario, requirement, content, evaluation, or product language.
- Provide both "conservative credible version" and "stronger version if evidence can be supplied" when appropriate.

For portfolio outputs:

- Start from a real user problem.
- Define target users, scenarios, pain points, product goals, MVP scope, AI capability, interaction flow, success metrics, risks, and iteration plan.
- Include model uncertainty, content safety, hallucination, evaluation, and human-in-the-loop design where relevant.

For interview outputs:

- Use structured answers: conclusion first, then evidence, then AI PM relevance.
- Acknowledge transition background directly and turn it into a strength.
- Avoid memorized-sounding answers.
