# Resume Bullet Bank

Use these patterns to rewrite experience into AI PM-facing resume bullets.

## Formula

Use:

`Action + scenario/user + method + output/result`

If results are unavailable:

`Action + scenario/user + method + product relevance`

## Product-Facing Verbs

分析、拆解、定义、设计、梳理、验证、评估、优化、沉淀、协同、推动、迭代、搭建、制定、复盘。

## Background Translation Patterns

Academic research:

- Before: 研究某语言现象。
- After: 围绕自然语言表达中的语义边界与使用场景进行分析，沉淀可用于文本分类、意图识别和输出质量评估的规则意识。

Teaching:

- Before: 负责语文教学。
- After: 面向不同学习水平的学生设计教学路径、练习任务和反馈机制，具备将用户目标拆解为内容结构与产品流程的能力。

Thesis:

- Before: 完成语言学论文。
- After: 基于文献、语料和案例分析完成研究课题，形成问题定义、资料整理、假设验证和结构化表达能力，可迁移至 AI 产品需求分析与评估设计。

Content writing:

- Before: 写文章、改稿。
- After: 针对不同受众与使用场景制定内容结构、表达风格和质量标准，适合参与 AI 写作、内容生成和文本优化类产品设计。

Interview or user research:

- Before: 做访谈。
- After: 通过访谈与文本材料分析识别用户需求、行为动机和场景痛点，输出结构化洞察与产品机会点。

## AI PM Resume Sections

Use these sections when relevant:

- 求职目标
- 教育背景
- AI 产品项目 / 作品集
- 实习 / 项目 / 教学 / 研究经历
- 产品与 AI 技能
- 语言与内容能力

## Credibility Guardrails

- Use "参与设计", "完成原型", "制定评估方案", "设计 Prompt", "分析竞品" when the user has project evidence.
- Avoid "负责模型训练", "优化算法", "搭建底层架构" unless the user truly did those things.
- If metrics are missing, ask for possible evidence:人数、材料数量、完成周期、反馈、产出数量、准确率、使用频次、作品链接.
