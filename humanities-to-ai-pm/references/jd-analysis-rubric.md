# JD Analysis Rubric

Use this when a user provides an AI product manager job description.

## Output Structure

1. Role summary
   - Identify product type, industry, user group, seniority, and whether the role is application PM, platform PM, data PM, or operations-heavy PM.

2. Capability extraction
   - Product capabilities.
   - AI/technical capabilities.
   - Industry/domain capabilities.
   - Collaboration capabilities.
   - Evidence the resume must show.

3. Match analysis
   - Strong matches from humanities/language/education/content background.
   - Partial matches that need stronger wording or portfolio evidence.
   - Gaps that need补强.

4. Resume keyword plan
   - Extract 8-15 keywords to reflect in resume and portfolio.

5. Application strategy
   - Recommend whether to apply now, apply after resume adjustment, or first build a portfolio project.

## Role Type Signals

Application AI PM:

- Keywords: LLM application, chatbot, AI assistant, prompt, workflow, scenario, user experience, Copilot.
- Strong fit for humanities backgrounds when the domain is language, education, content, knowledge, or service.

RAG/knowledge base PM:

- Keywords: knowledge base, enterprise search, document Q&A, retrieval, embedding, chunking, citation, accuracy.
- Strong fit if the user can show information architecture, content organization, or document workflow experience.

AI education PM:

- Keywords: tutoring, learning path, homework correction, assessment, teaching assistant, content generation.
- Strong fit for education and language backgrounds.

AI platform PM:

- Keywords: model platform, API, MLOps, inference, deployment, data pipeline, observability.
- Usually less suitable unless the user has technical project evidence.

Data/algorithm PM:

- Keywords: recommendation, ranking, model training, feature engineering, AB test, data strategy.
- Requires evidence of data analysis and technical collaboration.

## Red Flags

- Do not recommend a role as high-fit if it requires model training, distributed systems, or deep data engineering and the user has no evidence.
- Do not hide gaps. Convert gaps into portfolio tasks or study goals.
